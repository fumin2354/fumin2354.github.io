gitの使い方

基本的に小さい作業の単位でコミットを行い、ある程度作業がひと段落した時にプッシュをするのが一般的です。
コミットの作業がわかりやすいように、コミットメッセージを残しておくと、ログを追っていく時に役立ちます。

github-pageの作り方

1. ユーザ名.github.io という名前のリポジトリを作成する。
※例えば GitHub のユーザ名(アカウント名)が hoge の場合には hoge.github.io で作成。
2. そこに .html や .md (マークダウン) ファイルを Push する。
公開URLは https://ユーザ名.github.io/ と成ります。
使用例：https://fumin2354.github.io/

github-pageの更新の仕方

1. リポジトリを自身のアカウントに Fork してローカルにクローンを作成します
2. npm i コマンドで開発環境をインストールします
3. 作業用ブランチを切ります
4. pages フォルダー以下の該当ファイル (.md) を更新します
5. ファイルを更新したら、以下のビルド コマンドを実行します
6. コマンドを実行すると dist フォルダーに html に変換されたページ ファイル (ヘッダ部とコンテンツ部で分かれます) が作成されます (※これは無視して結構です)
7. その後に自動的にページ フォルダーが作成され、フォルダー内に完成版のページ ファイル (index.html) が作成されます
8. ビルドが完了したら npm run test を実行すると自動でテスト用 Web サーバーとブラウザーが立ち上がります (サーバーを停止するには [Ctrl] + C を押してください)
9. ページ レビューを行って問題がなければ EsriJapan/arcgis-dev-resources の master ブランチにプルリクエストしてください
10. 管理者 (@fumin) がマージをして gh-pages ブランチに更新内容を反映します
11. ArcGIS for Developers 開発リソース集 のページに反映され更新作業は完了です
